## v1.1.0

## Summary

Upgrades the Terraform Control action to commercial-grade PR reporting with a consolidated comment architecture. All plan/pre-check output from a single workflow now lands in one PR comment using named section blocks (`<!-- section:{id} -->`), eliminating per-action comment clutter.

## Changes

- **New input `comment-section-id`** — section identifier for the consolidated PR comment block. When set, the action upserts only its named section inside the shared `<!-- crosswave-terraform-report -->` comment rather than posting a standalone comment. Auto-derived from the working-directory slug when empty.
- **New outputs `plan_error_count` / `plan_error_file`** — structured error extraction on plan exit code 1. Replaces the expensive TRACE re-run anti-pattern with an awk-based pass over the existing plan log; writes clean `Error:` blocks to a temp file.
- **Commercial-grade plan summary** — emoji status line (🟢/🟡/🔴), resource counts table (Add / Change / Destroy), collapsible error details block (`<details>`), Checkov findings section, and a full-plan diff expander — all rendered into the consolidated comment section.
- **Section-replacement comment manager** — the `Terraform PR comment` github-script step now reads the existing consolidated comment, splices only the current section by ID, updates the timestamp header, and size-guards the body to 65 kB. Creates a new consolidated comment when none exists.
- **Removed TRACE re-run** — the fallback that re-executed `terraform plan` with `TF_LOG=TRACE` on failure has been removed. Errors are now extracted from the already-captured plan log without a second Terraform execution.

## Why

AL-010: PR comment threads were cluttered with separate comments from plan, pre-check, and matrix jobs. Consolidating into a single comment with named sections gives engineers one place to review Terraform state per PR. The TRACE re-run was a correctness hazard (different execution context) and a CI cost sink; structured extraction is equivalent and free.

## Usage

Pass `comment-section-id` to distinguish sections from concurrent matrix jobs:

```yaml
- uses: crosswave-technology/ActionLibrary-Terraform-Control@v1.1.0
  with:
    working-directory: terraform/
    command: plan
    comment-on-pr: "true"
    comment-section-id: plan-${{ inputs.account_id }}
```

Callers that omit `comment-section-id` continue to work — the section ID is derived from the working-directory slug automatically.
